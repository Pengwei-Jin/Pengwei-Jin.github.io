初学正则表达式

Posted on 2021-05-22 Edited on 2021-05-26 In notebook

正则表达式学习笔记

一、元字符

符号 描述
. 英文句号，匹配任意单个字符（不包括换行符）
[ ] 方括号[ ]内指定一个无序字符集，匹配字符集内的任意字符，用连字符指定字符集范围，例如 [w-zc] 匹配w、x、y、z、c；[ ]中的句号表示句号而不是任意单个字符
[^ ] 否定的字符集，匹配除了字符集内的任意字符
* 匹配在*之前的出现大于等于0次的字符，例如[a-z]*匹配一行中所有以小写字母开头的字符串
+ 匹配在+之前的出现大于等于1次的字符
? 标记?之前的字符为可选，即出现0或1次
{n,m} 限定一个或一组字符重复出现的次数小于等于n大于等于m，{n,}表示出现大于等于n次，{n}表示出现n次，不支持{,m}
( ) 特征标群，圆括号( )中包含的内容会被看成一个整体，例如(ab)*匹配连续出现大于等于0次的ab，ab*匹配a和之后出现的大于等于0次的b形成的字符串；( )中包含的内容会被单独记录下来供以后使用，如果不希望被单独记录，可使用(?: )
| 或运算符
\ 转义字符，用于匹配一些特殊字符[ ] ( ) { } . * + ? ^ $ \ / | `
^ 锚点，用来检查匹配的字符串是否在被匹配字符串的开头
$ 锚点，用来检查匹配的字符串是否在被匹配字符串的结尾

二、转义字符

符号 描述
\b 匹配单词边界，如果\b位于匹配的字符串的开始，它在单词的开始处查找匹配项,如果位于匹配的字符串的结尾，它在单词的结尾处查找匹配项，例如cd\b匹配abcd efg的cd，但是不匹配abcdef gh的cd
\B 匹配非单词边界
\w 匹配所有字母数字，等同于[a-zA-Z0-9_]
\W 匹配所有非字母数字，即符号，等同于[^\w]
\d 匹配数字，等同于[0-9]
\D 匹配非数字，等同于[^\d]
\s 匹配所有空白字符
\S 匹配所有非空白字符
\f 匹配一个换页符
\n 匹配一个换行符
\r 匹配一个回车符
\t 匹配一个制表符
\v 匹配一个垂直制表符

三、零宽度断言（前后预查）

符号 描述
?= 正先行断言，表示第一部分表达式之后必须跟着?=定义的表达式，返回结果中只包含满足匹配条件的第一部分表达式；定义一个正先行断言需要使用(?= )的格式，例如he(?=fat)匹配hefat的he，但是不匹配hecat的he
?! 负先行断言，筛选其后不跟随着断言中定义格式的匹配结果；定义一个负先行断言需要使用(?! )的格式
?<= 正后发断言，筛选其前跟随着断言中定义格式的匹配结果；定义一个正后发断言需要使用(?<= )的格式，例如(?<=he)fat匹配hefat的fat，但是不匹配sefat的fat
? 负后发断言，筛选其前不跟随着断言中定义格式的匹配结果；定义一个负后发断言需要使用(?的格式

四、标志

符号 描述
i 忽略大小写，例如/he/i匹配he、He、hE、HE
g 全局搜索，即不仅仅返回第一个匹配结果，而是返回全部匹配结果
m 多行修饰符，锚点元字符只会检查被匹配字符串的开头或者结尾，多行修饰符m可以使其在每行的开头和结尾生效

五、贪婪匹配与惰性匹配

*、+都是贪婪的，会尽可能多的匹配文字。在它们后面加上一个 ?就可以实现非贪婪或者最小匹配，例如对于字符串 abcabca，a.*a的匹配结果是 abcabca，a.*?a的匹配结果是 abca。

详细内容请参考http://www.regular-expressions.info/
